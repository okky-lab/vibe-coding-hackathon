## 목표

- 참가자가 아이디어를 실행 가능한 산출물로 전환하는 경험을 얻습니다.
- 문서 기반 협업으로 의사결정 맥락을 투명하게 남깁니다.
- 발표 가능한 수준의 데모 품질을 확보합니다.

## 결과물

최종 제출물은 아래 3가지로 구성됩니다.

1. 문제 정의 및 사용자 가치 설명 문서
2. 실행 가능한 제품 또는 프로토타입
3. 발표 자료 또는 데모 영상/링크

## 운영 원칙

- 빠른 검증: 완벽함보다 빠른 학습과 검증을 우선합니다.
- 명확한 기록: 팀의 주요 결정은 문서로 기록합니다.
- 협업 우선: 개인 성과보다 팀 단위 완주를 장려합니다.
