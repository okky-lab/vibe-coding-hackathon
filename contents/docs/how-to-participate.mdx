## 1. 팀 구성

- 팀은 기획/개발/디자인 역할 균형을 권장합니다.
- 1인 1팀을 권장하며, 2인 1팀으로 참여할 수 있습니다.

## 2. 주제 선정

- 실사용 문제를 명확히 정의합니다.
- 타깃 사용자와 문제의 우선순위를 정리합니다.

## 3. 빌드 진행

- 문서로 MVP 범위를 먼저 잠급니다.
- 행사 시간동안 진행할 MVP 범위를 문서로 기록해 오셔야 합니다.
- 구현 중 변경된 의사결정은 즉시 문서에 반영합니다.

## 4. 제출 준비

최종 제출 시 아래 항목을 포함해 주세요.

- 제품 링크 또는 실행 방법
- 문제 정의 문서
- 데모 설명(3분 이내 권장)
- 팀 소개 및 역할
- 최종 제출 시 아래의 스킬을 사용하여 제출하셔야 합니다.

```bash
add-skill @okky-lab/skills/apply
```

## 5. 발표

- 문제, 해결 방식, 핵심 데모, 다음 단계 순서로 발표합니다.
- 질의응답을 위해 주요 트레이드오프를 준비합니다.
- 팀당 발표시간은 3~5분이 주어지며 최고의 작품으로 발표합니다.
